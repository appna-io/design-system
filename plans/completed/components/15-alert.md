# Phase 15 — `<Alert />`

> Status: **Completed** · Depends on: Phase 6 (Button) · Blocks: future `<Toast>` (which reuses Alert's surface and the `_shared/iconForColor` helper extracted here)

## Objective

Ship the canonical inline status banner — `<Alert />` — for info / success / warning / danger /
neutral messages, with optional title, leading icon, action area, and dismissable interaction.

Alert is the **first component in the DS where the leading icon is selected by color**. Doing that
once (here) means Toast and future status surfaces (Banner, Inline-error) reuse the same
`iconForColor()` helper — no duplicated icon-mapping logic.

This is also the **first DS component with an exit animation** (Motion-driven dismissal).

---

## What This Component Proves

- An icon-by-state convention can live in a single helper used by multiple components.
- Motion can drive a controlled exit (`AnimatePresence`) without polluting the consumer's render tree.
- The variant system tolerates **constrained color sets** (Alert's `color` is a 5-of-7 subset — primary/secondary excluded).
- `<Alert.Title>` + `<Alert.Description>` + `<Alert.Action>` subpart pattern (subset of Card's compound model) reads cleanly when only some parts are used.

---

## Public API

```tsx
import { Alert } from 'apx-ds';

<Alert>Your changes have been saved.</Alert>

<Alert
  variant="soft"                  // 'solid' | 'outline' | 'soft' (default) | 'inline'
  color="info"                    // 'info' (default) | 'success' | 'warning' | 'danger' | 'neutral'
  size="md"                       // 'sm' | 'md' | 'lg'
  icon={<CustomIcon />}           // overrides the auto-selected icon for the color
  hideIcon={false}                // hides the leading icon entirely
  closable={false}                // shows a × button on the trailing end
  onClose={() => setOpen(false)}  // fired when dismissed
  open={open}                     // controlled visibility — pair with onClose
  defaultOpen={true}              // uncontrolled

  role="status"                   // 'status' | 'alert' — overrides the auto role; info=status, warning/danger=alert

  className=""
  style={{}}
  sx={{}}
>
  <Alert.Title>Saved successfully</Alert.Title>
  <Alert.Description>Your draft was saved to the cloud at 3:42 PM.</Alert.Description>
  <Alert.Action>
    <Button size="sm" variant="ghost">Undo</Button>
  </Alert.Action>
</Alert>
```

### Prop Decisions

- **5 colors, not 7** — `primary` and `secondary` are brand colors, not status semantics. Alerts speak about *what happened* (`info`, `success`, `warning`, `danger`) or *here's a note* (`neutral`). Forcing brand colors leads to UI drift. (Theme overrides can still add them per-project.)
- **`color="info"` is the default**, not `primary` — Alerts default to "general note", not "brand-aligned".
- **`role` auto-selects** based on `color`: `info` / `success` / `neutral` → `role="status"` (polite live region); `warning` / `danger` → `role="alert"` (assertive). Consumers override only when they know what they're doing.
- **`closable` adds a built-in × button** rather than asking consumers to wire it. Same convention as Badge's `removable`.
- **`open` + `onClose`** lets consumers run an exit animation (handled inside via `AnimatePresence`); when `open` flips to `false`, Alert animates out before unmount.

## Variants — Designed Inline

Four variants:

| Variant   | Background                       | Border                       | Text + icon                | When to use                                        |
| --------- | -------------------------------- | ---------------------------- | -------------------------- | -------------------------------------------------- |
| `solid`   | `bg-<color>`                     | `border-transparent`         | `text-<color>-contrast`    | Critical announcements ("System down").            |
| `outline` | `bg-bg-paper`                    | `border-<color>` 1px         | `text-<color>` accent + body `text-fg-default` | Brand-light contexts; pairs with Card.    |
| `soft`    | `bg-<color>-subtle`              | `border-<color>/30`          | `text-<color>` accent + body `text-fg-default` | **Default.** Conventional toast/banner.|
| `inline`  | `bg-transparent`                 | `border-transparent` (border-left-4 colored bar) | `text-<color>` accent + body `text-fg-default` | Inline form validation; minimal chrome. |

## Component Sketch

```tsx
'use client';
import { forwardRef, warn } from '@apx-ds/engine';
import { useThemedClasses } from '@apx-ds/theme';
import { AnimatePresence, motion } from 'motion/react';
import { iconForColor } from '../_shared/iconForColor';
import { alertRecipes } from './Alert.recipe';
import { alertMotion } from './Alert.motion';
import type { AlertProps } from './Alert.types';

const ROLE_BY_COLOR = {
  info: 'status', success: 'status', neutral: 'status',
  warning: 'alert', danger: 'alert',
} as const;

export const Alert = forwardRef<HTMLDivElement, AlertProps>(function Alert(props, ref) {
  const {
    variant, color = 'info', size,
    icon, hideIcon,
    closable, onClose,
    open: openProp, defaultOpen = true,
    role,
    className, style, sx,
    children,
    ...rest
  } = props;

  const [open, setOpen] = useControllableState({ prop: openProp, defaultProp: defaultOpen, onChange: (v) => { if (!v) onClose?.(); } });

  const { className: cls, style: rootStyle } = useThemedClasses({
    recipe: alertRecipes.root,
    componentName: 'Alert',
    slot: 'root',
    props: { variant, color, size, className, sx, style },
  });

  const Icon = hideIcon ? null : (icon ?? <DefaultIcon color={color} />);
  const ariaRole = role ?? ROLE_BY_COLOR[color];

  return (
    <AnimatePresence initial={false}>
      {open ? (
        <motion.div
          ref={ref}
          {...alertMotion}
          className={cls}
          style={rootStyle}
          role={ariaRole}
          data-variant={variant}
          data-color={color}
          {...rest}
        >
          {Icon ? <span aria-hidden="true" className="shrink-0 mt-0.5">{Icon}</span> : null}
          <div className="flex-1 min-w-0">{children}</div>
          {closable ? (
            <button
              type="button"
              aria-label="Dismiss"
              onClick={() => setOpen(false)}
              className="shrink-0 ms-2 -me-1 -mt-1 inline-flex items-center justify-center size-6 rounded hover:bg-black/10 dark:hover:bg-white/10 focus-visible:outline-none focus-visible:ring-1 focus-visible:ring-current"
            >
              <XIcon className="size-3.5" />
            </button>
          ) : null}
        </motion.div>
      ) : null}
    </AnimatePresence>
  );
}, 'Alert');

function DefaultIcon({ color }: { color: NonNullable<AlertProps['color']> }) {
  const Icon = iconForColor[color];
  return <Icon />;
}

const AlertTitle = forwardRef<HTMLDivElement, AlertTitleProps>(function AlertTitle(props, ref) {
  const { className, style, sx, children, ...rest } = props;
  const { className: cls, style: rootStyle } = useThemedClasses({
    recipe: alertRecipes.title, componentName: 'Alert', slot: 'title', props: { className, sx, style },
  });
  return <div ref={ref} className={cls} style={rootStyle} {...rest}>{children}</div>;
}, 'Alert.Title');

// AlertDescription, AlertAction follow the same shape — direct passthrough recipe applications.
```

Exported via `index.ts`:

```ts
export const Alert = Object.assign(AlertRoot, {
  Title: AlertTitle,
  Description: AlertDescription,
  Action: AlertAction,
});
```

## Motion

```ts
// Alert.motion.ts
export const alertMotion = {
  initial: { opacity: 0, y: -8, height: 0 },
  animate: { opacity: 1, y: 0, height: 'auto' },
  exit:    { opacity: 0, y: -8, height: 0 },
  transition: { duration: 0.18, ease: [0.4, 0, 0.2, 1] as const },
};
```

`height: 'auto'` ↔ `height: 0` is the trick that makes the surrounding layout reflow smoothly when
the Alert dismisses. Motion handles this — no manual measurement.

`prefers-reduced-motion`: Motion's built-in `useReducedMotion()` flips to `{ duration: 0 }` — we
re-export the same hook from the engine, no extra import.

## Accessibility

- Auto-`role`: `info`/`success`/`neutral` → `role="status"` (polite); `warning`/`danger` → `role="alert"` (assertive). Consumers override only when intentional.
- Icon is decorative (`aria-hidden`); the body text + role provide the meaning.
- `Alert.Title` is a `<div>` by default (not a heading) — same reasoning as Card.Header. Consumers wrap in `<h3>` when needed.
- `closable` button is keyboard-reachable; Enter + Space activate.
- Exit animation does not move focus — focus returns to wherever it was. If the Alert was focused, focus moves to the document body (browser default).
- axe-core: zero violations.

---

## Animation / Interactions

- Enter/exit: Motion-driven (`AnimatePresence`). Honors `prefers-reduced-motion` via the engine's `useReducedMotion()`.
- Bundle cost: Motion is already loaded for Button's spinner — Alert adds no new dependencies.
- Closable hover: CSS, identical pattern to Badge's removable button.

### Plan deviations (logged for future maintainers)

- **Icons sourced from `lucide-react` instead of inline SVGs.** `lucide-react` is already a
  hard dep used by Button; building five custom SVGs would have duplicated the
  tree-shakeable map for zero gain. The `_shared/icons/` subfolder mentioned in the plan
  was not created.
- **Five recipes, not four.** Added `alertCloseRecipe` for the dismiss button so its
  hover-tint reacts to the active variant via the `group/alert` CSS variable. Plan
  embedded the close styling inline — promoting it to a recipe keeps theme overrides
  per-slot (`Alert.styleOverrides.close`) symmetric with Title / Description / Action.
- **Motion easing**. The engine's `transitionTokens.ease.standard` is a CSS
  `cubic-bezier(…)` string, but Motion's JS runtime only accepts a named keyword or a
  4-tuple. `Alert.motion.ts` parses the bezier into a tuple at module load.
- **`controlBase` / `useFormFieldA11y` not consumed.** Alert is not a form control. The
  Phase 7 `_shared/` files were left untouched.
